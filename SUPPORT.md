# Support

## Usage questions

Use GitHub Discussions when available. Describe the repository revision, chosen
options, environment, expected result, actual result, and a minimal reproduction.

## Bugs and feature proposals

Use the corresponding GitHub issue form. Issues without enough information to
reproduce or evaluate may be converted to a discussion or closed.

## Security

Follow [SECURITY.md](SECURITY.md). Never disclose a suspected vulnerability in
an issue, discussion, or pull request.

This community project provides no guaranteed response time or private support.
